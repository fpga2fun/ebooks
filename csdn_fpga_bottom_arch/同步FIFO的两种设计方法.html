1、什么是FIFO？

        FIFO（First In First Out） 是一种先进先出的数据缓存器，在逻辑设计里面用的非常多。它是一种存储器结构，被广泛应用于芯片设计中。FIFO由存储单元队列或阵列构成，第一个被写入队列的数据也是第一个从队列中读出的数据。

        FIFO 设计可以说是逻辑设计人员必须掌握的常识性设计。FIFO 一般用在隔离两边读写带宽不一致，或者位宽不一样的地方。 在 FPGA 设计，使用 FIFO 一般有两个方法，第一个方法是直接调用官方的 FIFO IP，另外一个方法是自己设计 FIFO 控制逻辑。当然了，我们学会设计FIFO，并不一定是真的需要自己造轮子，只是说作为从业人员我们要了解相关的设计方法，毕竟自己造的轮子不一定能跑不是。

        为了更好地了解FIFO，可以把FIFO形象化地比作一条管道，管道中的球可以被看做写入FIFO的数据。FIFO的写操作和读操作过程如下所示：

        上图展示了FIFO的读写过程：

  • 在时刻①处：管道是空的，这个时候没有球进入，也没有球流出。即FIFO为空（empty），没有执行写操作，也没有执行读操作。

  • 在时刻②处：球（data1）被推入了管道中，管道不是空的了。即FIFO执行了写操作（data1被写入FIFO），没有执行读操作。

  • 在时刻③处：球（data2）被推入了管道中，此时管道满了（其最大容量为两个球）。即FIFO为满（full），执行了写操作（data2被写入FIFO），没有执行读操作。

  • 在时刻④处：没有球被推入管道中，球（data1）被从管道中拉出了。即FIFO执行了读操作（data1被从FIFO中读出），没有执行写操作。

  • 在时刻⑤处：没有球被推入管道中，球（data2）被从管道中拉出了，此时管道空了。即FIFO执行了读操作（data2被从FIFO中读出），没有执行写操作，此时FIFO为空。

        因为data1比data2更早进入FIFO，所以data2从FIFO中出来的也更早，这就是FIFO先入先出的含义。上图可能存在一些不完整和容易引起歧义的地方：

（1）是否需要FIFO被写满后，从可以读走数据？

        时刻3的操作很容易让人误解，是不是要数据把FIFO填满，才可以从中读走数据？

        不是，只要FIFO中有数据（即不是空的状态），就可以从中读走数据，比如下图中所示。上一个时刻FIFO中有数据data1，在下一个时刻没有新数据被写入FIFO，同时当前的数据data1被从FIFO中读出。

（2）是否同一时刻只能对FIFO进行一种操作，即不可同时读写？

        上面的过程很容易让人误解，是不是同一时刻只能对FIFO进行一种操作，即不可同时读写？

        不是。FIFO的读写操作是相对独立。只要FIFO中有数据（即不是空的状态），就可以从中读走数据；同样的，只要FIFO没有满，就可以往其中写入数据。比如下图中所示。上一个时刻FIFO中有数据data1（即不空也不满），在下一个时刻有新数据data2写入FIFO，同时当前的数据data1也被从FIFO中读出。

2、FIFO有什么用？

        在FPGA设计中，FIFO常常被用来实现如下功能：

  • 当输入数据速率和输出数据速率不匹配时，作为临时存储单元。例如，CPU可以先将数据写入FIFO，然后继续做其他工作，设备可以很方便地从FIFO中读取数据。再如因特网控制器，它将从网络接收来的数据存入FIFO，后端DMA（Direct Memory Access，直接存储器访问）控制器（位于PCIe或者PCI接口电路中）从FIFO中读取数据，然后写入系统存储器。

  • 用于不同时钟域之间的同步。实际应用中，数据将不得不从一个时钟域进入另一个时钟域，此时FIFO不仅作为临时数据存储单元，也起到数据同步的作用。

  • 输入数据路径和输出数据路径之间数据位宽不匹配时，可用于数据位宽调整电路。

        FIFO最常见的应用是缓冲读写瞬时速度不匹配的问题。比如下图中的发送和接收的例子，发送端的瞬时速率是10M/s，而接收端的瞬时速率是5M/s，如果没有FIFO来缓冲，那么接收端肯定是无法完整地接收发送端的数据的，从而会造成数据丢失。

        如果在二者之间有FIFO来缓冲，那发送端就可以用自己的速率来发送，而接收端也可以用自己的速率来接受。

        需要注意的是，FIFO是一个用来缓冲的存储器件，而不是用来单纯作为存储的。它不像你的U盘，你可以把电影存到U盘，然后看的时候打开就行了。使用FIFO是有限制条件的，那就是发送端的带宽和接收端的带宽要相等。

        如果写带宽大于读带宽，那不管FIFO的深度有多大，总是会有写满的时候，然后发送端的新数据就会丢失；如果写带宽小于读带宽，那不管FIFO的深度如何，FIFO总是很快就会空了，然后接收端就无法读到新的数据。

        你可能注意到了，上文中描述发送端和接受端的数据传输能力时我用了 “瞬时速率” 这个词，这是因为在实际情况中，发送和接收都是不是匀速的，更多的时候，是在某个时间段，发送一大段数据（这个就是瞬时速率，这种方式叫突发传输BURST），然后接下来停止传输一段时间，而接收端可能是匀速的，而且速度要比这段时间的平均速度慢，所以此时才需要把这一大段的数据缓存到FIFO，让接收端慢慢读出。

3、同步FIFO的设计

        根据读写两侧时钟是否一致，可以将FIFO分为两类：

  • 同步FIFO：读写时钟一致

  • 异步FIFO：读写时钟不一致

        同步 FIFO 有一个时钟信号，读和写逻辑全部使用这一个时钟信号，异步 FIFO 有两个时钟信号，读和写逻辑用的各种的读写时钟。

        因为同步FIFO读写时钟一致，所以没有异步FIFO的CDC问题，设计更简单，当然，同步FIFO也可以被看做是异步FIFO的一种特殊形式。实际使用中异步FIFO才是更常用的模块，因为它的读写时钟可以不一致，可以实现不同时钟域的信号传输，这更符合实际，也更能发挥FIFO的作用。本文只讨论如何实现同步FIFO，异步FIFO的实现下篇文章再介绍。

        FIFO 与普通存储器 RAM 的区别是没有外部读写地址线，使用起来非常简单，但缺点就是只能顺序写 入数据，顺序的读出数据，其数据地址由内部读写指针自动加 1 完成，不能像普通存储器那样可以由地址线决定读取或写入某个指定的地址。 FIFO 本质上是由 RAM 加读写控制逻辑构成的一种先进先出的数据缓冲器。

        FIFO 的常见参数如下：

  • FIFO 的宽度：即 FIFO 一次读写操作的数据位数

  • FIFO 的深度：指的是 FIFO 可以存储多少个 N 位的数据（如果宽度为 N）

  • 满标志（full/almost full）：FIFO 已满或将要满时由 FIFO 的状态送出的一个信号，以阻止 FIFO 的写操作继续向 FIFO 中写数据而造成溢出（overflow）

  • 空标志（empty/almost empty）：FIFO 已空或将要空时由 FIFO 的状态送出的一个信号，以阻止 FIFO 的读操作继续从 FIFO 中读出数据而造成无效数据的读出（underflow）

  • 读时钟：读操作所遵循的时钟，在每个时钟沿来临时读数据（同步FIFO中与写时钟一致）

  • 写时钟：写操作所遵循的时钟，在每个时钟沿来临时写数据（同步FIFO中与读时钟一致）

        同步FIFO的设计架构如下：

        因为读/写控制逻辑类似，所以实际上就是三个部分：

  • 读读/写控制逻辑：分别生成读指针和写指针。写指针总是指向下一个将要被写入的单元，复位时，指向第 1 个单元(编号为 0) ；读指针总是指向当前要被读出的数据，复位时，指向第 1 个单元(编号为 0) 。

  • 状态生成逻辑：主要是生成FIFO的空信号（empty）和满信号（full），当FIFO被写满时生成满信号，当FIFO内没有数据时生成读信号。

  • RAM读写逻辑：根据读指针/写指针的值，对相应的RAM单元进行读操作/写操作。

        FIFO 设计的关键是产生可靠的 FIFO 读写指针和生成 FIFO“空”/“满”状态标志。 当读写指针相等时，表明 FIFO 为空，这种情况发生在复位操作时，或者当读指针读出 FIFO 中最后一 个字后，追赶上了写指针时，如下图所示：

        当读写指针再次相等时，表明 FIFO 为满，这种情况发生在，当写指针转了一圈，折回来(wrapped around) 又追上了读指针，如下图：

        可见读写指针可以在读写使能有效时，每时钟周期+1，而如何产生可靠的“空”/“满”信号则成了同步FIFO设计的重点。下面有两种解决方法：

3.1、计数器法

        构建一个计数器，该计数器(fifo_cnt)用于指示当前 FIFO 中数据的个数：

  • 复位时，该计数器为0，表示此时FIFO中的数据个数为0

  • 当读写使能信号均有效且full != 0，且empty != 0时，说明此时FIFO又读又写（非空非满），则计数器不变，表示FIFO中的数据个数无变化

  • 当写使能有效且 full != 0，则 fifo_cnt +1，表示写操作且 FIFO 未满时候，FIFO 中的数据个数增加了 1

  • 当读使能有效且 empty != 0，则 fifo_cnt -1，表示读操作且 FIFO 未空时候，FIFO 中的数据个数减少了 1

  • fifo_cnt =0 的时候，表示 FIFO 空，需要设置 empty=1；fifo_cnt = fifo的深度 的时候，表示 FIFO 现在已经满，需要设置 full=1

3.1.1、Verilog代码

        有了思路以后，接下里就一步一步手撕代码：

（1）输入输出端口

        把FIFO的数据位宽和深度设置成了参数化形式，方便复用。写端口设计了满信号和几乎满信号；读端口设计了空信号和几乎空信号。

  1. module	sync_fifo_cnt
  2. #(
  3. 	parameter   DATA_WIDTH = 8  ,						// FIFO数据位宽
  4.     parameter   DATA_DEPTH = 16 						// FIFO深度
  5. )
  6. (
  7. // 系统端口
  8. 	input							clk			,		// 系统时钟
  9. 	input							rst_n		,		// 低电平有效的复位信号
  10. // 写端口					
  11. 	input	[DATA_WIDTH-1:0]		wr_data		,   	// 写入的数据
  12. 	input							wr_en		,   	// 写使能信号，高电平有效
  13. 	output							full		,		// 满标志，高电平表示当前FIFO已被读空
  14. 	output							almost_full	,		// 几乎满标志，高电平表示当前FIFO快被读空
  15. // 读端口		
  16. 	input							rd_en		,   	// 读使能信号，高电平有效	
  17. 	output	reg	[DATA_WIDTH-1:0]	rd_data 	,		// 读出的数据
  18. 	output							empty		,		// 空标志，高电平表示当前FIFO已被写满	
  19. 	output							almost_empty		// 几乎空标志，高电平表示当前FIFO快被写满	
  20. );

（2）参数/reg/wire定义

        把所有变量的位宽都参数化，这样的表示更规范。同时定义读地址/写地址，FIFO数据的个数计数器，和一个RAM来作为存储数据的介质。

  1. integer i;												// 循环变量	
  2.  
  3. // 参数定义
  4. localparam ADDR_WIDTH = $clog2(DATA_DEPTH);				// 读地址/写地址位宽，（$clog2是以2为底取对数）
  5. localparam CNT_WIDTH  = ADDR_WIDTH + 1;					// FIFO个数计数器的位宽，比地址的位宽多1位
  6. localparam AFULL  	  = DATA_DEPTH - 1;					// 几乎满阈值，设定为还差1个满就是几乎满。例如16深度的FIFO中有了15个数据就是几乎满	
  7. localparam AEMPTY  	  = 0 + 1;							// 几乎空阈值，设定为还差1个空就是几乎空。例如FIFO中还剩1个数据就是几乎空					
  8. // reg定义
  9. reg [DATA_WIDTH - 1 : 0]	mem [DATA_DEPTH - 1 : 0];	// 用二维数组RAM实现FIFO
  10. reg [ADDR_WIDTH - 1 : 0]	wr_addr;					// 写地址
  11. reg [ADDR_WIDTH - 1 : 0]	rd_addr;					// 读地址
  12. reg	[CNT_WIDTH  - 1 : 0]	data_cnt;					// FIFO中的数据个数，比地址的位宽多1位。以深度16的FIFO为例，其最大的个数为16，所以表示位宽需要5位

（3）读写控制逻辑

        读使能有效且非空时，读地址+1；写使能有效且非满时，写地址+1。注意因为本模块设计成了只支持深度为 2^N，所以地址在到了最大值后会自动回卷到0，这样做的好处是节省了面积（对比逻辑）。同时因为FPGA中最好使用BRAM来构建FIFO，而BRAM的深度都是 2^N，所以FIFO的深度最好也要设计成 2^N（这样可能会浪费BRAM面积，但是时序性能会更好）。

  1. /***************************读写控制逻辑************************************************/
  2. // 读操作，更新读地址
  3. always @ (posedge clk or negedge rst_n) begin
  4. 	if (!rst_n)
  5. 		rd_addr <= {(ADDR_WIDTH){1'b0}};
  6. 	else if (!empty && rd_en)					// 读使能有效且非空
  7. 		rd_addr <= rd_addr + 1'd1;				// 深度仅支持 2^N，这样地址会自动跳转到0，节省了面积
  8. end												// （续上）因为FPGA中构建FIFO的BRAM，其深度也是 2^N，所以这样使用是效率最高的
  9.  
  10. // 写操作,更新写地址
  11. always @ (posedge clk or negedge rst_n) begin
  12. 	if (!rst_n)
  13. 		wr_addr <= {(ADDR_WIDTH){1'b0}};
  14. 	else if (!full && wr_en)					// 写使能有效且非满
  15. 		wr_addr <= wr_addr + 1'd1;				// 深度仅支持 2^N，这样地址会自动跳转到0
  16. end

（4）RAM读写逻辑

读使能有效且非空时，从对应的RAM单元读出数据；写使能有效且非满时，将数据写入对应的RAM单元。复位时进行初始化，使用了for语句，放心，这是可以综合的。

  1. /***************************RAM读写逻辑************************************************/
  2. // RAM写操作
  3. always @ (posedge clk or negedge rst_n) begin
  4. 	if (!rst_n)												
  5. 		for(i=0;i<DATA_DEPTH;i=i+1)
  6. 			mem[i] <= {(DATA_WIDTH){1'b0}};		// 复位时先对RAM进行初始化，全部设置为0
  7. 	else if (!full && wr_en)					// 写使能有效且非满
  8. 		mem[wr_addr] <= wr_data;				// 把数据写入对应的RAM单元
  9. end
  10.  
  11. // RAM读操作
  12. always @ (posedge clk or negedge rst_n) begin
  13. 	if (!rst_n)												
  14. 		rd_data <= 	{(DATA_WIDTH){1'b0}};	
  15. 	else if (!empty && rd_en)					// 写使能有效且非满
  16. 		rd_data <= mem[rd_addr];				// 把数据写入对应的RAM单元
  17. end

（5）状态生成逻辑

根据FIFO中的数据个数生成对应的状态指示标志。同时根据读写使能信号，对计数器进行更新。

  1. /***************************状态生成逻辑************************************************/
  2. // 更新计数器
  3. always @ (posedge clk or negedge rst_n) begin
  4. 	if (!rst_n)
  5. 		data_cnt <= 0;
  6. 	else begin
  7. 		case({wr_en,rd_en})							// 拼接读写使能信号进行判断
  8. 			2'b00:data_cnt <= data_cnt;				// 不读不写
  9. 			2'b01:	                               	// 仅仅读
  10. 				if(data_cnt != 0)				   	// fifo没有被读空
  11. 					data_cnt <= data_cnt - 1'b1;   	// fifo个数-1
  12. 			2'b10:                                 	// 仅仅写
  13. 				if(data_cnt != DATA_DEPTH)         	// fifo没有被写满
  14. 					data_cnt <= data_cnt + 1'b1;	// fifo个数+1
  15. 			2'b11:data_cnt <= data_cnt;				// 读写同时
  16. 			default:;                              	
  17. 		endcase
  18. 	end
  19. end
  20.  
  21. // 依据计数器状态更新指示信号
  22. // 依据不同阈值还可以设计半空、半满 、几乎空、几乎满
  23. assign full = (data_cnt == DATA_DEPTH) ? 1'b1 : 1'b0;		// 满信号
  24. assign almost_full = (data_cnt == AFULL) ? 1'b1 : 1'b0;		// 几乎满信号
  25.  
  26. assign empty = (data_cnt == 0) ? 1'b1 : 1'b0;				// 空信号
  27. assign almost_empty = (data_cnt == AEMPTY) ? 1'b1 : 1'b0;	// 几乎空信号
  28.  
  29. endmodule

3.1.2、Testbench及仿真结果

        接下来编写脚本对源码进行测试：

  • 例化1个深度为8，位宽为8的同步FIFO

  • 先对FIFO进行写操作，直到其写满，写入的数据为随机数据

  • 然后对FIFO进行读操作，直到其读空

  • 然后对FIFO写入4个随机数据后，然后同时对其进行读写操作

  1. `timescale 1ns/1ns	//时间单位/精度

  3. //------------<模块及端口声明>----------------------------------------
  4. module tb_sync_fifo_cnt();
  5.  
  6. parameter   DATA_WIDTH = 8 ;			//FIFO位宽
  7. parameter   DATA_DEPTH = 8 ;			//FIFO深度
  8.  
  9. reg						clk			;
  10. reg						rst_n		;
  11.  
  12. reg	[DATA_WIDTH-1:0]	wr_data		;

  21.  
  22.

3.2、高位扩展法（指针法）

        在深度为8的FIFO中，需要3bit的位宽来分别指示读写地址3'b000-3'b111这8个地址。若将地址指针扩展1bit变成4bit的地址，那么地址表示区间则变成了4'b0000-4'b1111。假设不看最高位的话，后面3位的表示区间仍然是3'b000-3'b111，也就意味着最高位可以拿来作为指示位。

  • 当最高位不同且其他位相同，则表示读指针或者写指针多跑了一圈，而显然不会让读指针多跑一圈（多跑一圈读啥？），所以可能出现的情况只能是写指针多跑了一圈，与就意味着FIFO被写满了

  • 当最高位相同且其他位相同，则表示读指针追到了写指针或者写指针追到了读指针，而显然不会让写指针追读指针（这种情况只能是写指针超过读指针一圈），所以可能出现的情况只能是读指针追到了写指针，也就意味着FIFO被读空了

        这种方法有个缺点就是仅仅靠对比读写指针的值只能判断空状态和满状态，却无法判断几乎空状态和几乎满状态，因为几乎空和几乎满都是认为定义的，它和FIFO中的数据严格挂钩，所以如果要实现这个功能，就还需要设计一个计数器来对FIFO中的个数进行计数，实际上，这个计数器可以用写指针来减掉读指针。

3.2.1、Verilog代码

        有了思路以后，可以很容易的贴出Verilog代码（因为代码和上面的代码类似，就不分开贴了）：

  1. // *******************************************************************************************************
  2. // ** 作者 ： 孤独的单刀                                                   			
  3. // ** 邮箱 ： zachary_wu93@163.com
  4. // ** 博客 ： https://blog.csdn.net/wuzhikaidetb 
  5. // ** 日期 ： 2024/06/24
  6. // ** 模块 ： 使用高位扩展法（指针法）实现的同步FIFO模块。	
  7. // ** 功能 ： 1、同步FIFO，实现方式为计数器法；
  8. //			  2、可设置同步FIFO的数据位宽和深度（深度仅支持 2^N）；
  9. //            3、可输出空信号（empty）、满信号（full）、几乎空信号（almost_empty）和几乎满信号（almost_full）。
  10. // *******************************************************************************************************	
  11.  
  12.  
  13. module	sync_fifo_ptr
  14. #(
  15. 	parameter   DATA_WIDTH = 8  ,						// FIFO数据位宽
  16.     parameter   DATA_DEPTH = 16 						// FIFO深度
  17. )
  18. (
  19. // 系统端口
  20. 	input							clk			,		// 系统时钟
  21. 	input							rst_n		,		// 低电平有效的复位信号
  22. // 写端口					
  23. 	input	[DATA_WIDTH-1:0]		wr_data		,   	// 写入的数据
  24. 	input							wr_en		,   	// 写使能信号，高电平有效
  25. 	output							full		,		// 满标志，高电平表示当前FIFO已被读空
  26. 	output							almost_full	,		// 几乎满标志，高电平表示当前FIFO快被读空
  27. // 读端口		
  28. 	input							rd_en		,   	// 读使能信号，高电平有效	
  29. 	output	reg	[DATA_WIDTH-1:0]	rd_data 	,		// 读出的数据
  30. 	output							empty		,		// 空标志，高电平表示当前FIFO已被写满	
  31. 	output							almost_empty		// 几乎空标志，高电平表示当前FIFO快被写满	
  32. ); 
  33.  
  34. integer i;												// 循环变量	
  35.  
  36. // 参数定义
  37. localparam ADDR_WIDTH = $clog2(DATA_DEPTH);				// 真实读指针/真实写指针位宽，（$clog2是以2为底取对数）
  38. localparam AFULL  	  = DATA_DEPTH - 1;					// 几乎满阈值，设定为还差1个满就是几乎满。例如16深度的FIFO中有了15个数据就是几乎满			
  39. localparam AEMPTY  	  = 0 + 1;							// 几乎空阈值，设定为还差1个空就是几乎空。例如FIFO中还剩1个数据就是几乎空						
  40.  
  41. //reg define
  42. reg [DATA_WIDTH - 1 : 0]	mem [DATA_DEPTH - 1 : 0];	// 用二维数组RAM实现FIFO
  43. reg [ADDR_WIDTH : 0]		wr_ptr;						// 扩展写地址指针，位宽多1位（最高位不表示指针的实际值，只是用来比较的指示位）	
  44. reg [ADDR_WIDTH : 0]		rd_ptr;						// 扩展读地址指针，位宽多1位（最高位不表示指针的实际值，只是用来比较的指示位）	
  45.  
  46.  
  47. //wire define
  48. wire [ADDR_WIDTH - 1 : 0]	wr_ptr_true;				// 实际写地址指针，用来对RAM寻址
  49. wire [ADDR_WIDTH - 1 : 0]	rd_ptr_true;				// 实际读地址指针，用来对RAM寻址
  50. wire						wr_ptr_msb;					// 扩展写地址指针地址最高位
  51. wire						rd_ptr_msb;					// 扩展读地址指针地址最高位
  52. wire [ADDR_WIDTH : 0]		data_cnt;					// FIFO中的数据个数计数
  53.  
  54.  
  55. /*******************assign语句************************************/
  56. assign {wr_ptr_msb,wr_ptr_true} = wr_ptr;				// 将扩展指针的最高位与实际指针拼接
  57. assign {rd_ptr_msb,rd_ptr_true} = rd_ptr;				// 将扩展指针的最高位与实际指针拼接
  58. assign data_cnt = wr_ptr - rd_ptr;						// 因为有补码的存在，所以不会出现负数
  59.  
  60.  
  61. /***************************读写控制逻辑************************************************/
  62. // 读操作，更新读指针
  63. always @ (posedge clk or negedge rst_n) begin
  64. 	if (!rst_n)
  65. 		rd_ptr <= {(ADDR_WIDTH){1'b0}};
  66. 	else if (!empty && rd_en)					// 读使能有效且非空
  67. 		rd_ptr <= rd_ptr + 1'd1;				// 深度仅支持 2^N，这样地址会自动跳转到0，节省了面积
  68. end												// （续上）因为FPGA中构建FIFO的BRAM，其深度也是 2^N，所以这样使用是效率最高的
  69.  
  70. // 写操作,更新写指针
  71. always @ (posedge clk or negedge rst_n) begin
  72. 	if (!rst_n)
  73. 		wr_ptr <= {(ADDR_WIDTH){1'b0}};
  74. 	else if (!full && wr_en)					// 写使能有效且非满
  75. 		wr_ptr <= wr_ptr + 1'd1;				// 深度仅支持 2^N，这样地址会自动跳转到0
  76. end
  77.  
  78. /***************************RAM读写逻辑************************************************/
  79. // RAM写操作
  80. always @ (posedge clk or negedge rst_n) begin
  81. 	if (!rst_n)												
  82. 		for(i=0;i<DATA_DEPTH;i=i+1)
  83. 			mem[i] <= {(DATA_WIDTH){1'b0}};		// 复位时先对RAM进行初始化，全部设置为0
  84. 	else if (!full && wr_en)					// 写使能有效且非满
  85. 		mem[wr_ptr_true] <= wr_data;			// 把数据写入对应的RAM单元
  86. end
  87.  
  88. // RAM读操作
  89. always @ (posedge clk or negedge rst_n) begin
  90. 	if (!rst_n)												
  91. 		rd_data <= 	{(DATA_WIDTH){1'b0}};	
  92. 	else if (!empty && rd_en)					// 写使能有效且非满
  93. 		rd_data <= mem[rd_ptr_true];			// 把数据写入对应的RAM单元
  94. end
  95.  
  96.  
  97. /***************************状态生成逻辑************************************************/
  98.  
  99. // 依据计数器状态更新指示信号。依据不同阈值还可以设计半空、半满 、几乎空、几乎满
  100.  
  101. // 所有位相等时，读指针追到到了写指针，FIFO被读空。等价于 data_cnt 为0
  102. // assign empty = ( data_cnt == 0 ) ? 1'b1 : 1'b0;
  103. assign empty = ( wr_ptr == rd_ptr ) ? 1'b1 : 1'b0;	
  104.  
  105. // 当最高位不同但是其他位相等时，写指针超过读指针一圈，FIFO被写满。等价于 data_cnt 为FIFO深度
  106. // assign full = ( data_cnt == DATA_DEPTH ) ? 1'b1 : 1'b0;
  107. assign full  = ( (wr_ptr_msb != rd_ptr_msb ) && ( wr_ptr_true == rd_ptr_true ) )? 1'b1 : 1'b0;
  108.  
  109. assign almost_full  = ( data_cnt == AFULL  ) ? 1'b1 : 1'b0;	// 几乎满
  110. assign almost_empty = ( data_cnt == AEMPTY ) ? 1'b1 : 1'b0; // 几乎空
  111.  
  112. endmodule
