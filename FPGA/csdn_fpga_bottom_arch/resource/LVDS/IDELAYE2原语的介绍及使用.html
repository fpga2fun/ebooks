1、IDELAYE2是什么？

        IDELAYE2是FPGA的一个底层原语，它的功能主要是给输入的信号添加延迟。那为什么要给输入信号添加延迟呢？假如输入的数据分别为信号和时钟，由于这两个信号传输的频率可能都特别快，再加上它们的走线存在差别，所以它们进入FPGA时可能存在相位上的差别，如下所示：

        data信号在时钟的上升沿变化，此时的采样就很可能导致建立时间违例，从而引起亚稳态等功能错误。为了让信号在采样时刻即时钟上升沿保持稳定，可以将data信号延迟一段时间，让data信号在时钟的下降沿变化，而在时钟的上升沿则处于稳定状态，如下所示：

        原语IDELAYE2就可以完成这种功能，它最多可以延迟参考时钟的半个时钟周期，比如参考时钟是200MHz，它最多可以延迟78ps*32=2496ps约等于2.5ns，即半个时钟周期。

2、IDELAYE2的使用

        IDELAYE2是一款具有校准抽头分辨率的31抽头、环绕式延迟原语。FPGA的每个引脚都有一个IDELAYE2进行对应，可以对外部输入管脚的信号或者FPGA内部逻辑生成的信号进行延迟。

        IDELAYE2的原语框图如下，输入信号比较多，但是输出信号就两个，一个是延迟后的信号，另一个则是现在输出信号相对于输入信号的延迟系数，可用于对延迟效果的监控。

2.3、IDELAYE2的延时计算

        IDELAY2是一个可编程的31阶延迟原语，通过改变tap的值可以改变数据线的延时大小，其延时的分辨率与参考时钟频率f有关，延时的分辨率delay_resolution计算公式如下（f单位为Mhz）：                       

delay_resolution = 1/(32 * 2 * f)*1000000(ps)

  • 当f=200Mhz时，delay_resolution=78ps（注：计算出来为78.125ps，但会取整，为78ps）；

  • 同样，当f=300Mhz时，delay_resolution=52ps。

        延时的具体大小计算如下：

  • 当tap=0时不是延时就是0，因为IDELAYE2它作为一个底层器件，它自身也是具有延时的，此时的delay_time=600ps；

  • 当tap!=0时，delay_time=600ps + tap * delay_resolution

2.4、IDELAYE2的四种使用模式

        IDELAYE2的输入可以是来自FPGA管脚，此时要输入到信号IDATAIN；也可以是来自FPGA的内部逻辑，此时要输入到信号DATAIN。IDELAYE2有四种使用模式，它们通过参数IDELAY_TYPE来定义，分别是：

（1）IDELAY_TYPE = FIXED

        在固定延迟模式中，延迟值在配置时预设为参数 IDELAY_VALUE 确定的抽头数。 配置后，此值无法再更改。也就是说这个延迟值是通过参数 IDELAY_VALUE来定义的，在后续的使用过程中，这个延迟就不能再改变了。这种模式的灵活性比较低，需要一开始就测试好理想的延迟值才能用，在实际应用中较少用。

（2）IDELAY_TYPE = VARIABLE

        在可变延迟模式下，延迟值可以在配置后通过操纵控制信号 CE 和 INC 来改变。该模式的延迟初始值由参数IDELAY_VALUE确定，但是后续可以通过CE和INC信号递增或递减，各个信号的控制如下：

CLDCEINC延时值(TAP)
0XXXNo Change
11XXIDELAY_VALUE
100XNo Change
1011加1
1010减1
1000No Change

        下图显示了 IDELAYE2在该模式下的（IDELAY_VALUE = 0 ）时序图：

  • 时钟事件 1：在 C 的上升沿，检测到LD 为高电平，因为IDELAY_VALUE 为0，所以将输出 DATAOUT 设置为 Tap 0

  • 时钟事件 2：在 C 的上升沿检测到 CE 和 INC 同时为高电平（这表示增1操作），所以 输出从Tap 0 加1变化到Tap 1

  • 时钟事件 3：CE 和 INC 不再有效，输出保持不变

（3）IDELAY_TYPE = VAR_LOAD

        这个模式其实和VARIABLE模式差不多，只有一个区别：

  • VARIABLE模式在LD有效时，加载的初值来自参数IDELAY_VALUE，这意味着在该模式下加载的初值是固定的

  • VAR_LOAD模式在LD有效时，加载的初值来自输入信号CNTVALUEIN，这意味着在该模式下加载的初值是可变的

        下表为VAR_LOAD模式的控制信号真值表：

CLDCEINCCNTVALEIN延时值(TAP)
0XXXXNo Change
11XXCNTVALEINCNTVALEIN
100XXNo Change
1011X加1
1010X减1
10000No Change

        下图显示了 VAR_LOAD 模式下的 IDELAY 时序图：

  • 时钟事件 0：在 C 的上升沿，LD为低电平，且INC和CE也为低电平，所以此时没有输出

  • 时钟事件 1：在 C 的上升沿，LD 被检测为高电平，导致输出 DATAOUT 具有由 CNTINVALUE 定义的延迟，并将抽头设置更改为Tap 2。更新 CNTVALUEOUT 以表示新的抽头值。

  • 时钟事件 2：在 C 的上升沿捕获 CE 和 INC 上的脉冲。这表示增量操作。 输出从Tap 2 变化到Tap 3 ，CNTVALUEOUT 被更新以表示新的抽头值。

  • 时钟事件 3：在 C 的上升沿，LD 被检测为高电平，导致输出 DATAOUT 具有由 CNTINVALUE 定义的延迟，并将抽头设置更改为Tap 10。更新 CNTVALUEOUT 以表示新的抽头值。

（4）IDELAY_TYPE = VAR_LOAD_PIPE

        VAR_LOAD_PIPE模式和VAR_LOAD模式类似， 只是VAR_LOAD_PIPE模式内部多了一个寄存器，会将CNTVALEIN的值先存入寄存器，LD为高电平时，直接将寄存器中的值作为输出信号的延时数据。而LDPIPEEN信号为高电平时，将CNTVALUEIN的数据寄存到内部的寄存器。

3、IDELAYCTRL原语

        如果实例化 IDELAYE2 或 ODELAYE2 原语，则必须实例化 IDELAYCTRL 原语。

        IDELAYCTRL模块连续校准其区域内的各个IDELAY / ODELAY，以减少工艺、电压和温度变化的影响。IDELAYCTRL模块使用用户提供的REFCLK校准IDELAY和ODELAY，其框图如下所示：

        结构非常简单，只有2个输入+1个输出：

  • REFCLK：为 IDELAYCTRL 提供时间参考，以校准同一区域中的所有 IDELAY 和 ODELAY 模块。 该时钟必须由BUFG 或 BUFH驱动。 REFCLK 必须为 FIDELAYCTRL_REF ± 指定的 ppm 容差 (IDELAYCTRL_REF_PRECISION)，以保证指定的 IDELAY 和 ODELAY 分辨率 (TIDELAYRESOLUTION)。 REFCLK 可以直接从用户提供的源或 MMCM 提供，并且必须在BUFG上布线。

  • RST：复位输入引脚 (RST) 是高电平有效异步复位。 为确保 IDELAY 和 ODELAY 正常运行，IDELAYCTRL 必须在配置后复位并且 REFCLK 信号稳定。 需要一个复位脉冲宽度 TIDELAYCTRL_RPW。TIDELAYCTRL_RPW的值可以参考下图（59.28ns）。

  • RDY：指示何时校准特定区域中的 IDELAY 和 ODELAY 模块。 如果 REFCLK 保持高电平或低电平超过一个时钟周期，则 RDY 信号无效。 如果 RDY 置低，则必须复位 IDELAYCTRL 模块。

        RDY和RST的时序关系如下图所示：

        在复位信号RST有效过程中，RDY输出低电平，复位信号RST拉低经过TIDELAYCTRL_RDY(通过上上图可知该值为3.67us)后，RDY拉高，表示IDELAY延时有效，当检查到参考时钟信号REFCLK高电平持续时间过长时，RDY拉低，表示IDELAY延时无效。

4.1、FIXED模式的仿真

        FIXED模式的延迟是固定的，只能通过参数IDELAY_VALUE来设置，当IDELAY_VALUE=0时，延迟固定为600ps，IDELAY_VALUE不为0时，延迟=600ps+58ps×IDELAY_VALUE。TB如下，生成一个输入信号的脉冲，然后观察输入和输出信号的相位差。

  1. `timescale 1ns / 1ps
  2. ​
  3. module tb_IDELAYE2_FIXED();
  4. ​
  5. localparam IDELAY_TYPE  = "FIXED";  //FIXED, VARIABLE, VAR_LOAD, VAR_LOAD_PIPE
  6. localparam IDELAY_VALUE = 0;
  7. localparam PIPE_SEL     = "FALSE";
  8. ​
  9. reg             clk_in_50M  ;
  10. reg             reset       ;
  11. ​
  12. reg             ce          ;
  13. reg             inc         ;
  14. reg             ld          ;
  15. reg             din         ;
  16. reg             ldpipeen    ;
  17. reg             regrst      ;
  18. reg     [4:0]   cntvaluein  ;
  19. ​
  20. wire    [4:0]   cntvalueout ;
  21. wire            dout        ;
  22. ​
  23. initial begin
  24.     clk_in_50M = 1'b1;
  25.     forever #10 clk_in_50M = ~clk_in_50M;
  26. end
  27. ​
  28. initial begin
  29.     reset = 1'b1;
  30.     repeat(10)begin
  31.         @(posedge clk_in_50M);  
  32.     end
  33.     reset = 1'b0;
  34. end
