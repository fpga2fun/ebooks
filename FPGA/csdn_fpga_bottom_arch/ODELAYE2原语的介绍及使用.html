1、什么是ODELAYE2？

        在前面的文章我们已经对IDELAYE2有了详细的了解了，本文就来讲解一下和IDELAYE2的作用和使用方法都非常类似的另一个原语：ODELAYE2。

        ODELAYE2和IDELAYE2非常类似，IDELAYE2是将输入到FPGA管脚的信号延迟一段时间；而ODELAYE2则是将要从FPGA管脚输出到外部电路的信号延迟一段时间，其示意图如下所示：

        要注意的一点是，ODELAYE2原语在HR Bank内是没有的，因为A7系列只有HR Bank，也就意味着A7系列不能使用ODELAYE2这个原语。

        ODELAYE2也一定要和IDELAYCTRL一起使用，这一点和IDELAYE2的使用方法是一致的。

2、ODELAYE2的使用

        ODELAYE2是一款具有校准抽头分辨率的31抽头、环绕式延迟原语。FPGA的每个引脚都有一个ODELAYE2进行对应，可以对要从FPAG输出到外部的信号进行延迟。

        ODELAYE2的原语框图如下，输入信号比较多，但是输出信号就两个，一个是延迟后的信号，另一个则是现在输出信号相对于输入信号的延迟系数，可用于对延迟效果的监控。

2.1、信号及参数设置

        ODELAYE2的多数信号都与IDELAYE2的信号含义一致，但由于ODELAYE2在输出端，只能对FPGA内部信号(普通信号或者时钟)进行延时，相比IDELAYE2原语，ODELAYE2缺少IDATAIN信号，多了一个CLKIN，当需要延时的信号是时钟信号，从CLKIN端口输入。其信号列表如下所示：

信号名I/O位宽含义
CI1当ODELAYE2工作在FIXED模式下，不需要时钟信号，其余模式必须提供时钟信号。
REGRSTI1仅VAR_LOAD_PIPE模式下使用的复位。
LD、CE、INCI1与IDELAYE2中含义一致。
CINVCTRLI1动态翻转时钟 © 极性。
CNTVALUEINI5使用这个值来动态更改延迟抽头系数，位宽为5，最大31。
CLKINI1被延时信号是时钟信号时，从该端口输入，必须是时钟缓冲器（BUFIO、BUFG 或 BUFR）的信号。
DATAINI1来自FPGA逻辑的数据输入。
LDPIPEENI1把输入的延时值加载到流水线中。
DATAOUTO1输出延迟后的信号，输出到IOB。
CNTVALUEOUTO5输出该模块当前工作的延时系数

        ODELAYE2的参数与IDELAYE2的参数完全一致，如下表2所示，不再赘述，详解参考IDELAYE2参数讲解。

参数名含义
ODELAY_TYPEFIXED固定延时模式，VAR_LOAD 动态加载抽头值模式，VAR_LOAD或VAR_LOAD _PIPE模式下，可以动态加载外部输入的值，并进行动态调整，一般使用VAR_LOAD模式。
ODELAY_VALUEFIXED、VAR_LOAD模式的延时初始值，VAR_LOAD、VAR_LOAD_PIPE模式时被忽略。
CINVCTRL_SEL是否动态翻转时钟信号C的极性。
DELAY_SRC需要延时的信号是内部的信号(DATAIN)，还是管脚输入的信号(IDATAIN)
HIGH_PERFORMANCE_MODE是否使用高性能模式减小抖动，会增加功耗。
PIPE_SELVAR_LOAD_PIPE 模式的时候使用
REFCLK_FREQUENCY设置参考时钟频率（单位：MHz），取值范围190210、290310、390~410，通常取200，与延时的分辨率有关。
SIGNAL_PATTERN延时的是时钟信号还是数据，不同类型信号路径上的抖动不同。

2.2、延迟计算

        ODELAYE2是一个可编程的31阶延迟原语，通过改变tap的值可以改变数据线的延时大小，其延时的分辨率与参考时钟频率f有关，延时的分辨率delay_resolution计算公式如下（f单位为Mhz）：                       delay_resolution = 1/(32 * 2 * f)*1000000(ps)

  • 当f=200Mhz时，delay_resolution=78ps（注：计算出来为78.125ps，但会取整，为78ps）；

  • 同样，当f=300Mhz时，delay_resolution=52ps。

        延时的具体大小计算如下如下：

  • 当tap=0时不是延时就是0，因为ODELAYE2它作为一个底层器件，它自身也是具有延时的，此时的delay_time=600ps；

  • 当tap!=0时，delay_time=600ps + tap * delay_resolution

2.3、四种使用模式

        ODELAYE2同样有四种使用模式，使用方法和IDELAYE2完全一致，这里不赘述了，请参考IDELAYE2一文。

  • FIXED

  • VARIABLE

  • VAR_LOAD

  • VAR_LOAD_PIPE

3.1、FIXED模式的仿真

        FIXED模式的延迟是固定的，只能通过参数ODELAY_VALUE来设置，当ODELAY_VALUE=0时，延迟固定为600ps，ODELAY_VALUE不为0时，延迟=600ps+58ps×ODELAY_VALUE。TB如下，生成一个输入信号的脉冲，然后观察输入和输出信号的相位差。

​

  1. `timescale 1ns / 1ps
  2. ​
  3. module tb_ODELAY2_test();
  4. ​
  5. localparam ODELAY_TYPE  = "FIXED";  //FIXED, VARIABLE, VAR_LOAD, VAR_LOAD_PIPE
  6. localparam ODELAY_VALUE = 0;
  7. localparam PIPE_SEL     = "FALSE";
  8. ​
  9. reg             clk_in_50M  ;
  10. reg             reset       ;
  11. ​
  12. reg             ce          ;
  13. reg             inc         ;
  14. reg             ld          ;
  15. reg             din         ;
  16. reg             ldpipeen    ;
  17. reg             regrst      ;
  18. reg     [4:0]   cntvaluein  ;
  19. ​
  20. wire    [4:0]   cntvalueout ;
  21. wire            dout        ;
  22. ​
  23. initial begin
  24.     clk_in_50M = 1'b1;
  25.     forever #10 clk_in_50M = ~clk_in_50M;
  26. end
  27. ​
  28. initial begin
  29.     reset = 1'b1;
  30.     repeat(10)begin
  31.         @(posedge clk_in_50M);  
  32.     end
  33.     reset = 1'b0;
  34. end
  35. ​
  36. initial begin
  37.     ce = 1'b0;
  38.     inc = 1'b0;
  39.     ld = 1'b0;
  40.     ldpipeen = 1'b0;
  41.     regrst = 1'b0;
  42.     cntvaluein = 4'b0;
  43.     din = 1'b0;
  44.     //@(negedge reset); //PLL输出稳定了
  45.     @(posedge u_ODELAY2_test.locked);   //PLL输出稳定了
  46.         
  47.     //生成一个输入脉冲
  48.     @(negedge clk_in_50M);din = 1'b1;
  49.     @(negedge clk_in_50M);din = 1'b0;
  50.     
  51.     @(negedge clk_in_50M);$stop;
  52. end
  53. ​
  54. ​
  55. //例化被测试模块
  56. ODELAY2_test #(
  57.     .ODELAY_TYPE    (ODELAY_TYPE    ),
  58.     .ODELAY_VALUE   (ODELAY_VALUE   ),
  59.     .PIPE_SEL       (PIPE_SEL       )
  60. )
  61. u_ODELAY2_test(
  62.     .clk_in_50M     (clk_in_50M     ),
  63.     .reset          (reset          ),
  64.     .ce             (ce             ),
  65.     .inc            (inc            ),
  66.     .ld             (ld             ),
  67.     .din            (din            ),
  68.     .ldpipeen       (ldpipeen       ),
  69.     .regrst         (regrst         ),
  70.     .cntvaluein     (cntvaluein     ),
  71.     .cntvalueout    (cntvalueout    ),
  72.     .dout           (dout           )   
  73. );
  74. ​
  75. endmodule

        首先复位MMCM，然后等待其输出稳定，再接着等待IDELAYCTRL输出稳定，然后观察输入和输出的相位差是否与预期一致。

        当延迟系数即ODELAY_VALUE设置为0，输入和输出的相位差理论上是600ps，下图是实际仿真结果，相位差也是600ps，和预期一致。

        将延迟系数即ODELAY_VALUE修改为10，输入和输出的相位差理论上是600ps+78*10ps=1380ps，下图是实际仿真结果，相位差也是1380ps，和预期一致。

3.2、VARIABLE模式的仿真

        VARIABLE模式的延迟需要在LD信号有效时从参数ODELAY_VALUE来装载，在CE有效时，根据INC的值来递增（INC=1）或递减（INC=0）。TB如下，生成多个输入信号的脉冲，然后观察输入和输出信号的相位差。

        仿真结果如下，首先生成了10个周期的同步复位信号来复位MMCM，一段时间后locked信号拉高表示MMCM输出稳定，接着RDY信号拉高，表示IDELAYCTRL信号输出也稳定了，然后对延迟进行调节。

        载入延迟的初始系数15，此时的理论延迟为600ps+15×78ps=1770ps，实际测得的延迟也为1770ps。

        CE和INC同时为高电平，表示将抽头即延迟系数+1，从15变成16，此时的理论延迟值为600ps+78ps×16 = 1848ps，实际测得的延迟也为1848ps。

        CE为高电平，INC为低电平，表示将抽头即延迟系数-1，从16变成15，此时的理论延迟值为600ps+78ps×15 = 1770ps，实际测得的延迟也为1770ps。

3.3、VAR_LOAD模式的仿真

        VAR_LOAD模式和VARIABLE模式非常类似，只是其装载的初值不是来自参数ODELAY_VALUE，而是来自输入信号CNTVALUEIN，这样一来更加灵活。同样的，在CE有效时，根据INC的值来递增（INC=1）或递减（INC=0）。TB如下，生成多个输入信号的脉冲，然后观察输入和输出信号的相位差。

​

  1. `timescale 1ns / 1ps
  2. ​
  3. module tb_ODELAY2_test();
  4. ​
  5. localparam ODELAY_TYPE  = "VAR_LOAD";   //FIXED, VARIABLE, VAR_LOAD, VAR_LOAD_PIPE
  6. localparam ODELAY_VALUE = 15;
  7. localparam PIPE_SEL     = "FALSE";
  8. ​
  9. reg             clk_in_50M  ;
  10. reg             reset       ;
  11. ​
  12. reg             ce          ;
  13. reg             inc         ;
  14. reg             ld          ;
  15. reg             din         ;
  16. reg             ldpipeen    ;
  17. reg             regrst      ;
  18. reg     [4:0]   cntvaluein  ;
  19. ​
  20. wire    [4:0]   cntvalueout ;
  21. wire            dout        ;
  22. ​
  23. initial begin
  24.     clk_in_50M = 1'b1;
  25.     forever #10 clk_in_50M = ~clk_in_50M;
  26. end
  27. ​
  28. initial begin
  29.     reset = 1'b1;
  30.     repeat(10)begin
  31.         @(posedge clk_in_50M);  
  32.     end
  33.     reset = 1'b0;
  34. end
  35. ​
  36. initial begin
  37.     ce = 1'b0;
  38.     inc = 1'b0;
  39.     ld = 1'b0;
  40.     ldpipeen = 1'b0;
  41.     regrst = 1'b0;
  42.     cntvaluein = 4'b0;
  43.     din = 1'b0;
  44.     //@(negedge reset); //PLL输出稳定了
  45.     @(posedge u_ODELAY2_test.locked);   //PLL输出稳定了
  46.         
  47.     //装载初值
  48.     @(negedge clk_in_50M);ld = 1'b1;cntvaluein = 5'd20;

  54.     //抽头系数+1

  62.     @(negedge clk_in_50M);ce = 1'b1;inc = 1'b0;
  63.     @(negedge clk_in_50M);ce = 1'b0;inc = 1'b0;
  64.     //生成一个输入脉冲
  65.     @(negedge clk_in_50M);din = 1'b1;
  66.     @(negedge clk_in_50M);din = 1'b0;

  69. end
  70. ​

  83.     .ld             (ld             ),
  84.     .din            (din            ),
  85.     .ldpipeen       (ldpipeen       ),

  89.     .dout           (dout           )

3.4、VAR_LOAD_PIPE模式的仿真

        VAR_LOAD_PIPE模式和VAR_LOAD模式类似， 只是VAR_LOAD_PIPE模式内部多了一个寄存器，会将CNTVALEIN的值先存入寄存器，LD为高电平时，直接将寄存器中的值作为输出信号的延时数据。而LDPIPEEN信号为高电平时，将CNTVALUEIN的数据寄存到内部的寄存器。接下来的仿真我们就观察一下如何载入初值即可，后续的+1/-1操作就不观察了。TB如下：

​

  1. `timescale 1ns / 1ps
  2. ​
  3. module tb_ODELAY2_test();
  4. ​
  5. localparam ODELAY_TYPE  = "VAR_LOAD_PIPE";// FIXED, VARIABLE, VAR_LOAD, VAR_LOAD_PIPE
  6. localparam ODELAY_VALUE = 10;
  7. localparam PIPE_SEL     = "TRUE";
  8. ​
  9. reg             clk_in_50M  ;
  10. reg             reset       ;
  11. ​
  12. reg             ce          ;
  13. reg             inc         ;
  14. reg             ld          ;
  15. reg             din         ;
  16. reg             ldpipeen    ;
  17. reg             regrst      ;
  18. reg     [4:0]   cntvaluein  ;
  19. ​
  20. wire    [4:0]   cntvalueout ;
  21. wire            dout        ;
  22. ​
  23. initial begin
  24.     clk_in_50M = 1'b1;
  25.     forever #10 clk_in_50M = ~clk_in_50M;
  26. end
  27. ​
  28. initial begin
  29.     reset = 1'b1;
  30.     repeat(10)begin
  31.         @(posedge clk_in_50M);  
  32.     end
  33.     reset = 1'b0;
  34. end
  35. ​
  36. initial begin
  37.     ce = 1'b0;
  38.     inc = 1'b0;
  39.     ld = 1'b0;
  40.     ldpipeen = 1'b0;
  41.     regrst = 1'b0;
  42.     cntvaluein = 4'b0;
  43.     din = 1'b0;
  44.     //@(negedge reset); //PLL输出稳定了
  45.     @(posedge u_ODELAY2_test.locked);   //PLL输出稳定了
  46.         
  47.     //装载初值5到内部寄存器
  48.     @(negedge clk_in_50M);ldpipeen = 1'b1;cntvaluein = 5'd5;
  49.     @(negedge clk_in_50M);ldpipeen = 1'b0;cntvaluein = 5'd5;    
  50.     
  51.     //间隔一段时间
  52.     @(negedge clk_in_50M);cntvaluein = cntvaluein + 1;
  53.     @(negedge clk_in_50M);cntvaluein = cntvaluein + 1;
  54.     @(negedge clk_in_50M);cntvaluein = cntvaluein + 1;
  55.     @(negedge clk_in_50M);cntvaluein = cntvaluein + 1;
  56.     
  57.     //装载初值
  58.     @(negedge clk_in_50M);ld = 1'b1;
  59.     @(negedge clk_in_50M);ld = 1'b0;
  60.     
  61.     //生成一个输入脉冲
  62.     @(negedge clk_in_50M);din = 1'b1;
  63.     @(negedge clk_in_50M);din = 1'b0;
  64.         
  65.     @(negedge clk_in_50M);$stop;
  66. end
  67. ​
  68. ​
  69. //例化被测试模块
  70. ODELAY2_test #(
  71.     .ODELAY_TYPE    (ODELAY_TYPE    ),
  72.     .ODELAY_VALUE   (ODELAY_VALUE   ),
  73.     .PIPE_SEL       (PIPE_SEL       )
  74. )
  75. u_ODELAY2_test(
  76.     .clk_in_50M     (clk_in_50M     ),
  77.     .reset          (reset          ),
  78.     .ce             (ce             ),
  79.     .inc            (inc            ),
  80.     .ld             (ld             ),
  81.     .din            (din            ),
  82.     .ldpipeen       (ldpipeen       ),
  83.     .regrst         (regrst         ),
  84.     .cntvaluein     (cntvaluein     ),
  85.     .cntvalueout    (cntvalueout    ),
  86.     .dout           (dout           )   
  87. );
  88. ​
  89. endmodule

        仿真结果如下所示，首先拉高LDPIPEN，把在CNTVALUEIN上的值5载入内部寄存器，然后拉高LD，把内部寄存器的值5作为真正的初值载入，并在CNTVALUEOUT上输出。理论上输入和输出的延迟至为600ps+78ps×5=990ps，实际测得的延迟值也为990ps。
